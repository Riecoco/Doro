# AI Disclosure Statement

During the development of Doro, AI tools were used as a support tool for planning, debugging, and understanding implementation choices. AI also assisted with generating code suggestions and implementation approaches, which I reviewed, adapted, and integrated into the application throughout the development process.

## How AI Was Used

AI was used for:

* Breaking down the original project proposal into a realistic final scope.
* Planning which features were most important for the deadline and grading rubric.
* Explaining Vue concepts such as props, emits, Pinia stores, component structure, and event forwarding.
* Helping reason through frontend component architecture, especially for the task panel, task items, quote modals, and timer store.
* Debugging issues related to Vue event flow, modal behavior, task editing, and API integration.
* Reviewing whether the project met the assignment requirements.

## My Own Work

The final implementation, project decisions, code integration, testing, and feature selection were done by me. I made the decisions about which features to keep, which features to postpone, and how the final application should behave.

## Reflection on AI Use

AI helped me work more efficiently, especially when I was stuck or needed help understanding a concept. However, I still had to adapt the advice to my own project, test the code, make design decisions, and ensure that the final application matched the assignment requirements.

AI was used as a learning and support tool, not as a replacement for understanding the project.
